# Roadmap

This roadmap is for the public readiness kit only. It is not a product implementation roadmap, standards roadmap, certification roadmap, conformance roadmap, patent-license roadmap, or production deployment roadmap.

## 0–30 days

- Publish public GitHub repository.
- Keep `make demo` and `make qa` clean on a fresh checkout.
- Invite scenario-card feedback from developers, buyers, auditors, researchers, civil society, and public-sector teams.
- Use `AAIF_FEEDBACK_INQUIRY.md` for feedback or working-group-fit outreach, not as a hosted-project submission.

## 30–90 days

- Add more scenario-card authoring examples.
- Improve developer ergonomics for the educational linter without turning it into production verification.
- Add public walkthrough videos or screenshots if useful.
- Collect public issues that clarify vocabulary or scenario-card quality.

## 3–6 months

- Evaluate whether a separately licensed true software slice should exist.
- Add external reviewers or maintainers only if they preserve the public/private boundary.
- Publish a public learning report on common HOLD/REFUSE patterns.

## 6–12 months

- Consider foundation, working-group, or technical-committee paths only if there is independent community activity, maintainership, adoption evidence, and clear license/governance fit.
- Do not represent the kit as an official hosted project, standard, conformance suite, certification kit, or reference implementation unless the relevant governance body has actually approved that status.
