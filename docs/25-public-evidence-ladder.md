# Public Evidence Ladder

Boundary: public education and readiness discussion only. Not a product implementation, standard, certification kit, compliance approval, patent license, or production allow/deny system.

The Public Evidence Ladder helps a team decide when a conversation should move from vocabulary to readiness review to separate conformance evidence.

| Level | Name | Public readiness question | Output |
|---:|---|---|---|
| 0 | Vocabulary only | Do participants distinguish access, permission, execution, and acceptance? | Shared language |
| 1 | Scenario-card discussion | Can the team discuss ACCEPT / HOLD / REFUSE using public scenario cards? | Tabletop notes |
| 2 | Readiness index report | Are boundary, evidence, freshness, replay, escalation, and procurement questions identified? | Educational readiness report |
| 3 | Procurement/RFP evidence request | Can the buyer ask vendors for action-level evidence without requesting private mechanisms? | Public RFP prompts |
| 4 | Conformance evaluation handoff | Has the team reached the point where separate conformance artifacts are needed? | Handoff to external artifact family |
| 5 | Independent verifier / third-party reproduction path | Does a separate verifier or third party reproduce the public target? | External evidence outside this kit |

The Public Readiness Kit stops at educational readiness and handoff. It does not contain production evidence objects, cryptographic binding, conformance vectors, verifier logic, certification rights, or implementation licenses.

## Use this ladder when

- a board asks whether agentic AI action risk is being framed at the right boundary;
- a buyer wants public-safe RFP questions;
- a builder wants to know whether they need an implementation review;
- an auditor wants to separate education from conformance evidence.

## Do not use this ladder as

- a certification program;
- a legal compliance mapping;
- a vendor approval process;
- a production safety case;
- a substitute for separate conformance artifacts.
