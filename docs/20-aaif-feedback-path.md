# AAIF Feedback Path

This kit should be approached as a public readiness and educational utility, not as an immediate hosted-project submission.

## Current posture

Use:

> free public readiness kit with a 5-minute educational scenario-card linter for developers

Do not use:

> free version, open-source implementation, reference implementation, standard, conformance kit, certification kit

## Outreach path

1. Publish the GitHub repository.
2. Make sure `make demo` and `make qa` pass on a clean checkout.
3. Use `AAIF_FEEDBACK_INQUIRY.md` to ask for feedback, working-group fit, or technical-committee sponsor guidance.
4. Consider hosted-project submission later only if there is independent community participation, appropriate governance, external maintainers, contributor base, adoption evidence, issue activity, and trademark/account donation readiness.
