# FAQ

## Is this an open-source Acceptance Plane implementation?

No. This is a public readiness kit. It opens public vocabulary, scenario cards, procurement questions, and educational material. It does not disclose or license production mechanisms.

## Why include a tool at all?

The tool checks whether public scenario cards are complete enough for discussion. It is intentionally non-production and does not verify real actions, issue receipts, authorize effects, certify systems, or implement controls.

## Is this a standard?

No. It is not a formal standard, product specification, API contract, compliance certification, or certification program.

## Does this grant a patent license?

No. The license covers copyrightable public materials only and does not grant patent rights, trademark rights, implementation rights, certification rights, compliance approval, or endorsement.

## Can a vendor claim compliance with this kit?

No. Vendors may say they have read or used the public readiness materials, but this kit does not create compliance, certification, approval, or conformance status.

## What should remain private or licensed?

Exact schemas, cryptographic binding, verifier APIs, enforcement pipelines, conformance suites, certificate registries, hardware adapters, domain implementations, private deployments, and patent claim maps.

## What is the simplest public message?

Access is not authority. Permission is not proof. Execution is not acceptance. Production action should not happen without evidence.
