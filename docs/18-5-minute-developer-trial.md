# 5-Minute Developer Trial

Goal: let a developer understand the public readiness concept without installing dependencies or touching production systems.

```bash
make demo
```

The demo checks three scenario cards and prints whether each card is complete enough for tabletop discussion.

This is not a production allow/deny decision. It is not certification, compliance approval, or production readiness.

## Why this matters

Developers can immediately see the difference between:

- model output,
- tool permission,
- runtime access,
- protected-system impact,
- action acceptance before impact.

The linter does not implement the Acceptance Plane. It teaches the public question.
