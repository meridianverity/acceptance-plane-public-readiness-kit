# Public Release Boundary Checklist

Use this checklist before publishing, tagging, archiving, presenting, or sharing updates to the public readiness kit.

## Must be present

- Public tagline: “Public readiness materials for action acceptance before impact.”
- Rights boundary: “Not a product implementation. Not a standard. No patent license. No certification right.”
- Canonical thesis citation.
- Patent and trademark notice.
- Public/private split language.
- QA report and manifest verification.

## Must not be present

- Production verifier code.
- Exact receipt or evidence schema.
- Cryptographic binding method.
- Non-bypassable enforcement adapter.
- Signed conformance vector corpus.
- Certificate registry implementation.
- Hardware-specific adapter or driver path.
- Patent claim chart or claim mapping.
- Partner-specific deployment material.
- Unpublished patent language.

## Naming discipline

Safe public names include “readiness kit,” “public profile,” “scenario corpus,” “RFP questions,” “tabletop guide,” and “public-interest covenant.” Avoid “standard,” “certification,” “certified,” “SDK,” “reference implementation,” and “conformance suite” for this public release.

## Release discipline

Publish the category. Open the readiness layer. Protect the mechanism.
