# Acceptance Plane Public Maturity Model

This maturity model is a public education tool. It is not a certification ladder, compliance status, procurement approval, safety approval, implementation profile, or standard.

## Level 0 — Logs after impact

The organization relies on broad access, broad permissions, and after-the-fact logs. It may reconstruct what happened later, but it cannot explain why a specific autonomous action was accepted before impact.

## Level 1 — Named action inventory

The organization can list high-consequence autonomous or semi-autonomous actions and the protected systems they may affect.

## Level 2 — Named acceptance boundary

The organization can point to where AI intention becomes protected-system consequence for each high-consequence action class.

## Level 3 — Evidence categories mapped

The organization maps public evidence categories for each high-consequence action class: action, actor, delegation, scope, target, policy state, runtime or environment, freshness, revocation, replay, approval state, consequence, and decision reason.

## Level 4 — Scenario-disciplined operations

The organization uses tabletop scenarios, procurement questions, and public ACCEPT / HOLD / REFUSE language to reason about action acceptance. It has documented hold/refuse paths and human escalation routes.

## Level 5 — Production evaluation readiness

The organization is ready to evaluate licensed, authorized, or standards-track production mechanisms for verifier-ready receipts, cryptographic binding, enforcement, conformance, or domain-specific gates. This public kit does not supply or license those mechanisms.

## Level 6 — Ecosystem accountability

Multiple organizations can communicate across procurement, audit, insurance, regulatory, and infrastructure boundaries using shared action-acceptance language. Level 6 is an ecosystem goal, not a claim made by this kit.

## Anti-inflation rule

Do not call any public maturity level “certified.” Do not imply that use of this kit proves safety, compliance, conformance, implementation quality, or authorization to deploy production acceptance controls.
