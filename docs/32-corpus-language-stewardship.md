# Corpus Language Stewardship

The v0.3.4 release keeps language polish as part of public trust. A public readiness corpus can pass structural checks and still lose credibility if repeated phrasing makes it look synthetic, careless, or overproduced.

This release therefore adds a narrow editorial gate for visible scenario-card scars:

- duplicate a duplicated `before` token phrasing;
- incorrect article use such as incorrect article before `incident`;
- incorrect article use such as incorrect article before `operator`.

The gate is intentionally modest. It does not judge whether a scenario is legally sufficient, operationally complete, compliant, production-ready, or allowed. It only protects the public educational corpus from avoidable wording defects that reviewers can see immediately.

## Stewardship boundary

This language QA is for public education and editorial stewardship only. It is not certification, compliance approval, procurement approval, conformance testing, production readiness, or an allow/deny decision.


## v0.3.3 learning-goal polish

v0.3.3 extended the language-polish gate to reject repeated reaches-before wording after an audit found ACCEPT-card learning goals with a visible editorial scar. The release kept the educational corpus intact while making that scar a blocker for future releases.
