Abstract

Visible abstract for citation and indexing.

Agentic artificial intelligence systems increasingly run inside confidential, accelerated, local, private, or hybrid execution environments. Hardware-rooted attestation can provide strong evidence about where and how a runtime executed, but attested execution is not itself sufficient authorization for every inference request, tool call, credential release, data egress event, dispatch, or regulated workflow step. This report introduces a dual-gate evidence rail for attested agentic AI: a deny-before-inference gate that controls whether an execution request may reach inference or a downstream action executor, and a permit-before-commit gate that controls whether a proposed external effect may be accepted at an effect boundary. The first gate binds an execution request to origin provenance, immutable constraint validation, distributed authorization, policy epoch state, and tamper-evident evidence logging before inference or downstream action. The second gate binds a proposed external-effect request to canonical action bytes, an action digest, nonce or challenge material, policy digest, epoch identifier, time-bounded validity, revocation status, and a machine-verifiable decision or permit receipt before commitment. Together, the two gates convert hardware-rooted execution posture into request-scoped and action-scoped authorization semantics without placing enforcement inside the token decode loop or accelerator kernel hot path. The contribution is a vendor-neutral control pattern for AI execution compliance: attested execution is necessary, but not sufficient; each request and each external effect needs fresh, scoped, replay-resistant, machine-verifiable authorization evidence.
